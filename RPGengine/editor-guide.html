Часть I. Основы

1. Введение: что вы создаёте

JSON RPG Editor — визуальный конструктор текстовой RPG на базе D&D 5e с опциональной поддержкой Pathfinder 2e (отдельные классы и родословные в том же файле данных). Вся игра — один game_data.json, в котором описаны:

  • Сцены — экраны с текстом, диалогами, кнопками и опционально боем.
  • Выборы — куда ведёт игрок, проверки навыков, скрытые ветки по условиям.
  • Расы и классы — создатель персонажа: сначала раса, затем класс; бонусы расы (ASI или PF2e Boosts), подсказки из ui_hints.
  • Умения и прогрессия — бой, ресурсы, уровни 1–10, ASI, концентрация, ячейки заклинаний.
  • Флаги и квесты — память сюжета (flags) и журнал со стадиями (questStages, questSet).
  • Репутация и торговля — числовые флаги rep_*, скидки в лавке.
  • Расходники-свитки — предметы с useAbility (огненный шар из инвентаря).
  • Карта мира — быстрые переходы между открытыми локациями.

Редактор (editor.html) меняет JSON. Игра (index.html) читает те же данные и запускает сюжет. Вам не нужно писать код, если вы используете встроенные механики: условия, квесты, бой, предметы, проверки навыков.

Новое в движке

2. Обновления движка (после первой версии руководства)

Краткий список механик, уже работающих в index.html и описанных ниже подробнее.

  • Визуальный конструктор условий — правила showIf/hideIf/states без ручного JSON (вкладка сцен, превью условий).
  • Квесты по стадиям — questSet на выборе, журнал «Текущие задания», правило «Стадия квеста» в условиях.
  • Ячейки заклинаний — у мага и паладина (с 2 уровня) ресурс отображается точками по кругам; у воина — «ярость» (очки энергии).
  • Расходники с умением — у предмета поле useAbility (ID из progression.abilities), например свиток огненного шара.
  • Репутация — числовые флаги rep_village и др.; влияние на цены; статусы «Вражда / Нейтралитет / Дружба / Герой» в сайдбаре; условия min / max на флаг.
  • Концентрация (D&D 5e) — одно длительное заклинание; проверка Телосложения при уроне; индикатор [C] в бою.
  • Зелье фокусировки — преимущество на проверки концентрации 1 час (лавка Джека, 22 зм).
  • Прогрессия до 10 уровня — при повышении выбор 1 умения из 2–3 вариантов; на 4, 8 и 10 уровнях — ASI (+2 к одной характеристике или +1 к двум).
  • Создатель персонажа — раса → класс → характеристики (Point Buy или PF2e Boosts) → навыки → бонусное умение 1 уровня.
  • Расы (D&D 5e) — вкладка 🧬 Расы: ASI, черты, языки; бонусы применяются после Point Buy (потолок 20).
  • Родословные (PF2e) — расы с system: "pf2e", фиксированные и свободные Boosts, обязательное наследие (heritages), ОЗ от расы.
  • Классы D&D — воин, маг, паладин, плут, жрец, следопыт; классы PF2e — pf2e_* с полем system: "pf2e".
  • Подсказки UI — блок ui_hints для классов и характеристик в создателе персонажа.
  • Дополнительные квесты — в демо-модуле: «Тень Люкорна» (lukorn_investigation), «Медальон Эльзы» (albert_locket).
  • Plugin API для special — SpecialSceneRegistry, JSON в plugins.specialScenes, свои обработчики в отдельных JS без правки engine.js.
Патч js/patch-progression.js при загрузке данных подставляет таблицы уровней 2–10, ASI и ячейки — правки в JSON можно дублировать в редакторе или оставить патчу.

Как данные попадают в игру

  1. Вы редактируете проект в editor.html.
  2. Нажимаете Сохранить JSON — скачивается game_data.json.
  3. Для теста: положите файл в data/ и выполните node scripts/sync-data-js.js или в игре нажмите Загрузить контент JSON и выберите файл.
  4. Для игроков: Экспорт игры — папка с index.html и уже вшитым js/data.js.

4. Запуск редактора и игры

Редактор

  1. Откройте editor.html в Chrome или Edge.
  2. Для кнопки Экспорт игры (запись папки на диск) удобен локальный сервер, например Live Server в VS Code — адрес вида http://127.0.0.1:5500/..., не file://.
  3. Нажмите Загрузить JSON и выберите data/game_data.json или свой экспорт.

Игра

  1. Откройте index.html.
  2. После правок в редакторе обновите данные (см. §3) или загрузите JSON вручную в игре.
  3. Сохранение прогресса игрока — кнопки в сайдбаре игры (браузерное хранилище).

Часть II. Создание игры

6. Создание игры с нуля: пошаговый план

  1. Новый проект — введите название игры.
  2. Мета — при PF2e-кампании укажите meta.system: "pf2e" (§11).
  3. Расы — хотя бы одна раса под выбранную систему (§20).
  4. Классы — минимум один класс с тем же system, что у проекта (§19).
  5. Предметы — оружие, зелья, ключи (ID понадобятся в наградах и условиях).
  6. Враги — если планируется бой.
  7. Сцены — цепочка от start: текст + выборы «Переход».
  8. Стартовый экран — сцены выбора класса с action: select_class:ИД и сцена имени с start_game.
  9. Квесты — создайте квест, стадии 0, 1, 2…; на выборах укажите questSet для смены стадии.
  10. Условия — конструктор «Показать если» на кнопках (флаги, предметы, стадия квеста).
  11. Карта — точки и mapLocation на сценах.
  12. Тема и звуки — по желанию.
  13. Проверить проект → исправить замечания.
  14. Экспорт игры или Сохранить JSON → пройти игру в index.html с нуля.

7. Практикум: мини-кампания «Пропавшая реликвия» (пошагово)

Ниже — полный цикл без кода: от пустого проекта до играбельной ветки с квестом, условием и наградой. Повторяйте действия в редакторе параллельно с чтением.

Шаг A. Подготовка данных

  1. Новый проект → название «Реликвия».
  2. Классы → класс hero, название «Герой», ОЗ 20, КД 14, ресурс «Энергия» max 2.
  3. Предметы → old_key (misc), relic (misc), heal_potion (consumable).
  4. Квесты → ID relic_quest, стадия 0: log «Священник просит вернуть реликвию», hint «Поговорите в храме»; стадия 1: hint «Найдите реликвию в склепе»; стадия 2: finish ✓, log «Реликвия возвращена».

Шаг B. Сцены сюжета

  1. Сцена class_pick — текст «Выберите путь», выбор select_class:hero.
  2. Сцена name_entry — кнопка start_game → temple (после создателя персонажа в игре).
  3. Сцена temple — текст «Священник ждёт у алтаря».
    • Выбор «Принять поручение» → crypt_entrance, блок questSet: квест relic_quest, стадия 0.
    • Выбор «Уйти» → village (без квеста).
  4. Сцена crypt_entrance — showIf: стадия квеста relic_quest = 0 или 1 (режим «Любое» с двумя правилами questStage, либо одна стадия 1 после принятия).
    • Выбор «В склеп» → crypt_inside только если нет предмета relic (правило «Нет предмета»).
  5. Сцена crypt_inside — skillCheck Восприятие DC 12: успех → флаг found_niche, предмет old_key; провал → остаётесь без ключа.
  6. Сцена relic_room — showIf: флаг found_niche = true И hasItem old_key. Выбор «Взять реликвию» → temple, questSet стадия 1, награда предмет relic на сцене (items).
  7. Сцена temple — второй выбор «Сдать реликвию» — showIf: hasItem relic + questStage 1; questSet стадия 2; золото 50.

Шаг C. Проверка

  1. На сцене temple откройте Превью видимости: без реликвии кнопка «Сдать» скрыта; с реликвией и стадией 1 — видна.
  2. Проверить проект → исправьте битые to.
  3. Карта сюжета — убедитесь, что нет изолированных узлов.
  4. Сохраните JSON → откройте игру → пройдите путь от класса до сдачи реликвии.
Этот практикум использует только формы редактора: квесты, questSet, конструктор условий, skillCheck. JSON понадобится только для consumable use, если хотите лечащее зелье.

9. Вкладка JSON: когда она нужна

Вкладка JSON показывает весь проект. Для сюжета она не обязательна. Используйте её, когда:

  • Нужно скопировать фрагмент сцены или квеста между проектами.
  • Поле ещё не вынесено в форму: repeatRewards, use у зелья, content у книги, ui_hints, statusEffects.
  • Ищете опечатку в ID по всему файлу (Ctrl+F).
После ручного редактирования JSON не нажимайте «Сохранить» в браузере на вкладке — данные живут в памяти редактора. Ошибка синтаксиса (лишняя запятая) может сломать экспорт. Проверяйте валидность и снова Загрузите JSON, если что-то пошло не так.

9.1 Диалоги на сцене

Кнопка + Добавить в секции диалогов. Каждая строка: Говорящий + Текст. В игре реплики идут после основного абзаца, отдельными блоками.

Пример диалога 
"dialogue": [
  { "speaker": "Марта", "text": "Опять чужаки… Чего вам?" },
  { "speaker": "Вы", "text": "Ищу дорогу к мельнице." }
]

Имя говорящего — произвольная строка; связь со вкладкой NPC не обязательна, но удобно писать те же имена, что в справочнике NPC.

10. Метаданные проекта

Слева внизу блок Проект → ✏️ Мета. Заполняются:

  • meta.title — название (заголовок окна при экспорте).
  • meta.author — автор.
  • meta.description — краткое описание.
  • meta.version — версия (при новом проекте «1.0»).
  • meta.system — активная система правил для создателя и боя (см. §11). Если поле пустое, движок выводит систему из содержимого JSON (наличие PF2e-классов/рас).

11. Системы правил (D&D 5e / Pathfinder 2e)

Один проект может содержать оба набора классов и рас, но в игре активна одна система — из meta.system:

Значение meta.systemПоведение
dnd5e (по умолчанию)Point Buy 27 очков, расы без system: "pf2e", классы без system: "pf2e", модификаторы D&D 5e.
pf2eAbility Boosts вместо Point Buy, обязательное наследие, классы и расы с system: "pf2e", расчёт ОЗ с учётом races.*.hp.
genericУпрощённые правила, если в проекте нет заклинаний (редко для готовых модулей).

11.1 Как разделить контент в JSON

  • Классы PF2e — ID вида pf2e_fighter, поле "system": "pf2e" в объекте класса.
  • Родословные PF2e — ID вида human_pf2e, поле "system": "pf2e" в объекте расы.
  • Классы и расы D&D — поле system отсутствует или не равно pf2e.
Не меняйте ID базовых классов демо (warrior, wizard, paladin) — на них завязаны сцены select_class. Новые классы D&D добавляйте отдельными ID (rogue, cleric, ranger).

11.2 Новый проект и шаблон PF2e

При создании проекта движок и редактор подставляют meta.system по содержимому. Для чистой PF2e-кампании задайте meta.system: "pf2e" в метаданных или используйте шаблон PF2e в редакторе (если доступен в вашей сборке).

Часть III. Сцены и повествование

12. Сцены: полный разбор полей

Сцена — один «экран» игры. ID сцены — латиница без пробелов (tavern_main, forest_path).

12.1 Как создать сцену

  1. Вкладка Сцены → слева + Новая сцена.
  2. Введите ID (например village_square).
  3. В диалогах укажите локацию (заголовок в игре) и текст.
  4. В форме сцены добавьте Выборы (см. §14).
Пример: простая сцена с двумя выходами 
{
  "location": "Деревенская площадь",
  "text": "Вы стоите у колодца. На севере тропа в лес, на востоке — таверна.",
  "choices": [
    { "text": "В таверну", "to": "tavern", "icon": "🍺" },
    { "text": "В лес", "to": "forest_gate", "icon": "🌲" }
  ]
}

12.2 Поля формы сцены

ПолеВ игре

13. Состояния локации (states)

Одна и та же сцена может выглядеть по-разному в зависимости от прогресса — без дублирования десятка сцен. Блок 🔄 Состояния локации появляется под текстом сцены.

Как это работает

  1. Движок проверяет states[] сверху вниз.
  2. Для каждого элемента вычисляется condition (визуальный конструктор — как у showIf).
  3. Первое истинное состояние подменяет: text, location, audio, choices, special.
  4. Если ни одно не подошло — используется default (галочка Fallback) или базовые поля сцены.
  5. Бой, флаги сцены, предметы при входе не меняются states — только отображение.
Пример: деревня до и после пожара 
"states": [
  {
    "id": "burning",
    "condition": {
      "all": [
        { "flag": "village_on_fire", "equals": true }
      ]
    },
    "text": "Деревня охвачена пламенем. Жители бегут к реке.",
    "choices": [
      { "text": "Помочь тушить", "to": "fight_fire", "icon": "🔥" }
    ]
  },
  {
    "id": "peaceful",
    "default": true,
    "text": "Тихая площадь. Пахнет свежим хлебом."
  }
]
  1. Откройте сцену → + Добавить в блоке состояний.
  2. Задайте ID состояния и Условие (condition) в конструкторе (например флаг = true).
  3. Заполните override-текст и при необходимости свои выборы.
  4. У одного состояния отметьте Fallback (default) — оно покажется, если другие условия ложны.

16. Смена стадии квеста на выборе (questSet)

Чтобы продвинуть квест без ручного флага, в карточке выбора есть блок 📜 При выборе — стадия квеста.

  1. Нажмите + При смене стадии квеста.
  2. Выберите квест и стадию из списков (стадии из вкладки Квесты).
  3. Игрок при нажатии этой кнопки получит новую стадию в журнале.
Пример: взять квест у NPC 
{
  "text": "Я помогу найти мельника",
  "to": "village_after_quest",
  "questSet": { "questId": "find_albert", "stage": "0" }
}
Пример: завершить квест (стадия с finish: true) 
{
  "text": "Сдать отчёт",
  "to": "reward_scene",
  "questSet": { "questId": "find_albert", "stage": "3" }
}
Связывайте видимость других кнопок с правилом Стадия квеста в showIf, а не с отдельными флагами quest_* — так проще сопровождать сюжет.

На выборе можно задать изменение репутации (в JSON вкладки сцены):

"reputation": { "rep_village": 5 }

При нажатии кнопки движок вызовет changeReputation и запишет в журнал при заметном изменении.

17.4 Репутация (rep_*)

В демо-модуле «Мельница на Тихой реке»:

  • Стартовое значение: startingFlags.rep_village: 0 в game_data.json.
  • Метаданные фракции: reputation.rep_village (название «Деревня», иконка).

  • Лавка Джека: цена × (1 − rep/100), скидка до 30%, наценка до 50%; при rep_village ≤ −20 — отказ в торговле (сцена jack_buy / special shop_jack).

В редакторе для условий числового флага пока удобнее вкладка JSON на выборе: "min": 10 или "max": -20 рядом с flag.

18. Квестовая система

Квесты живут во вкладке 📜 Квесты. В игре отображаются в Журнале заданий по полям log (текст записи) и hint (подсказка «что делать»).

18.1 Создание квеста

  1. + Новый квест → введите ID (find_albert).
  2. Заполните название и описание.
  3. + Стадия — стадии нумеруются 0, 1, 2… (в JSON ключи строки "0", "1").
  4. У каждой стадии: log, hint, галочка Завершение квеста (finish) на финальной стадии.
  5. Опционально: награда XP, скрытый квест (hidden).
Пример квеста в JSON (после сохранения из редактора)

  "hidden": false,
  "rewards": { "exp": 150, "gold": 25 },
  "stages": {
    "0": {
      "log": "Марта просит найти мельника Альберта.",
      "hint": "Поговорите с трактирщицей и осмотрите мельницу.",
      "finish": false
    },
    "1": {
      "log": "Вы нашли следы у амбара.",
      "hint": "Осмотрите амбар и погреб.",
      "finish": false
    },
    "3": {
      "log": "Альберт спасён. Деревня благодарна.",
      "hint": "",
      "finish": true
    }

}

18.2 Как продвигать квест в сюжете

Рекомендуемый способ: на выборе кнопки укажите questSet (§16). Стадия попадёт в state.questStages[questId].

Условия: на других кнопках используйте правило Стадия квеста в showIf — «квест X на стадии 1».

На карте: у точки worldMap можно задать показ только при нужной стадии квеста (вкладка Карта).

18.3 Скрытые квесты

Если у квеста hidden: true, он не виден в журнале, пока стадия не станет известной игроку (логика движка). Удобно для сюжетных сюрпризов.

18.4 Завершение и награды

Стадия с finish: true помечает квест завершённым. Награда rewards.exp / gold / items может выдаваться движком при переходе на финальную стадию (см. логику в engine.js для вашей версии).

Шесть играбельных классов без поля system (или с системой по умолчанию): warrior, wizard, paladin, rogue, cleric, ranger. Подсказки в создателе — ключи ui_hints.class_warrior … class_ranger.

19.2 Классы Pathfinder 2e

Четыре класса с "system": "pf2e": pf2e_fighter, pf2e_cleric, pf2e_rogue, pf2e_wizard. Отображаются только при активной PF2e-системе (§11).

Нужны сцены:

  2. Сцена ввода имени (в демо — отдельная логика).
  3. Кнопка start_game → переход на первую игровую сцену (start или свою).

20.1 Расы D&D 5e

В демо: human, elf, dwarf, halfling, dragonborn, tiefling. Основные поля:

  "name": "Человек",

20.2 Родословные PF2e

  • abilityBoosts — фиксированные ключи (str, free…); free игрок распределяет в создателе.

  2. Заполните имя, иконку, описание, бонусы и черты.
  3. Для PF2e добавьте хотя бы одно наследие в списке heritages.
  4. Сохраните JSON и проверьте создатель в index.html.

21. Создатель персонажа в игре

После select_class (класс уже выбран на сцене) открывается пошаговый мастер character-creator.js.

21.1 D&D 5e

  1. Раса — сетка доступных рас; отображаются бонусы ASI к характеристикам.
  2. Класс — подтверждение или смена (только классы текущей системы).
  3. Point Buy — 27 очков, характеристики 8–15; затем применяется ASI расы (макс. 20).
  4. Навыки — владения класса + bonusSkills расы.
  5. Умение 1 уровня — одно из level1BonusChoices.
  6. Имя и финализация.

21.2 Pathfinder 2e

  • Раса и наследие (heritage) обязательны.
  • Распределение фиксированных и свободных Boosts вместо Point Buy (потолок характеристик 18 на 1 уровне).
  • Классы только с system: "pf2e".

21.3 Подсказки (ui_hints)

В JSON можно задать тексты для панелей при наведении / выборе:

  • class_warrior, class_wizard, class_paladin, class_rogue, class_cleric, class_ranger — описание классов.
  • stat_str … stat_cha — пояснения характеристик.
  • Также: hp, ac, inventory, travel и др. для сайдбара игры.

Редактируются во вкладке JSON или при копировании из демо-модуля.

21.4 Умения и ОЗ при создании

  • Базовые умения — из classes.*.abilities (воин: Второе дыхание, Всплеск, Блок щитом; маг: снаряд, щит, пылающие руки…).
  • Дополнительное умение — одно из level1BonusChoices (патч patch-progression.js или JSON класса).
  • Активные черты расы (traits type active) могут попасть в список умений.
  • ОЗ: кость класса + мод. ТЕЛ (D&D) или логика PF2e с races.*.hp.

24. Прогрессия и уровни (1–10, ASI)

Вкладка 📈 Прогрессия и патч js/patch-progression.js задают поведение в игре.

  • maxLevel: 10, опыт: 0, 80, 180, 320, 500, 750, 1100, 1550, 2100, 2800 (пороги уровней).
  • На уровнях 2–10 у каждого класса в progression.levels — массив choices: 2–3 ID умений; игрок обязан выбрать одно в модальном окне.
  • На уровнях 4, 8 и 10 — флаг asi: true: сначала окно ASI (+2 к одной характеристике или +1 к двум), затем выбор умения (если есть choices).
  • Ячейки заклинаний мага/паладина и ярость воина обновляются по таблицам в патче при каждом уровне.
Пример: уровень 4 воина в JSON класса 
"4": {
  "asi": true,
  "choices": ["warrior_armor_mastery", "death_strike"]
}
Старые сохранения (например, 3 уровень) не теряют умения; новые выборы появятся только при следующем повышении уровня.

26. Бой: концентрация, ячейки, интерфейс

26.1 Ячейки заклинаний и ярость

В сайдбаре под ресурсом класса:

  • Маг — точки по кругам (1–5), тратятся при spellLevel умения.
  • Паладин — полукастер: ячейки с 2 уровня.
  • Воин — «Ярость» (очки энергии), тратятся по полю cost.

26.2 Концентрация

Умения с concentration: true (щит веры, удержание личности, зеркальный образ…):

  • Только одно активное заклинание; новое снимает старое.
  • При получении урона — спасброс Телосложения (DC 10 или половина урона).
  • В бою: метка [C] Название; при провале — «Концентрация потеряна!»
  • Концентрация сбрасывается в конце боя.

26.3 Статус-эффекты

Яд, кровотечение, оглушение, реген — вкладка данных statusEffects и эффект apply_status в умениях.

29. Специальные сцены (special) и Plugin API

Поле special у сцены запускает обработчик из реестра SpecialSceneRegistry (js/special-scenes.js) до обычного показа текста. Ядро engine.js менять не нужно: подключайте логику через JSON или отдельный скрипт.

29.1 Встроенные ID (демо-модуль)

  • haggle — торг с NPC.
  • shop_jack — лавка (цены с учётом rep_village).
  • jack_buy_* — покупка зелья, верёвки, припасов, свитка, зелья фокусировки.
  • forest_loot_check, barn_chest, attic — обыск/сундуки.
  • gear_*, boss_*, cellar_* — сюжет демо-модуля.
  • reset — сброс игры.

В редакторе все ID подсказываются в datalist поля special, включая записи из plugins.specialScenes.

29.2 JSON-плагины (plugins.specialScenes)

Без JavaScript: опишите мини-сцену в JSON. На сцене укажите "special": "dice_game" (тот же ID, что в плагине). После загрузки данных движок зарегистрирует обработчик автоматически.

Пример секции plugins в game_data.json 
"plugins": {
  "specialScenes": {
    "dice_game": {
      "label": "Мини-игра в кости",
      "text": "Трактирщик предлагает сыграть в кости. Ставка — 10 золотых.",
      "dialogue": [
        { "speaker": "Трактирщик", "text": "Ну что, рискнёшь?" }
      ],
      "choices": [
        { "text": "🎲 Играть (10 зм)", "to": "dice_game_roll", "showIf": { "goldMin": 10 } },
        { "text": "← Отказаться", "to": "tavern" }
      ]
    },
    "alchemy_station": {
      "label": "Алхимический станок",
      "text": "Старый алхимический стол. Колбы и реагенты покрыты пылью.",
      "flags": { "visited_alchemy": true },
      "items": ["empty_vial"],
      "choices": [
        { "text": "🔍 Осмотреть", "to": "alchemy_examine" },
        { "text": "← Уйти", "to": "laboratory" }
      ]
    }

}

Поддерживаемые поля конфига: label, text, dialogue, choices, flags, items (массив ID), gold (число). Встроенные ID из демо перезаписать JSON-ом нельзя (в консоли будет предупреждение).

29.3 Внешний JS (без правки engine.js)

Подключите свой файл после special-scenes.js в index.html / экспорте игры и зарегистрируйте обработчик:

SpecialSceneRegistry.register(
  'my_event',
  'Моё событие',
  function (engine, sceneId, scene) {
    engine.setText('Что-то произошло!');
    engine.setChoices([{ text: 'Дальше', to: 'village' }]);
    engine.saveGame();
  }
);

Либо укажите имя существующего метода GameEngine строкой (как встроенные): register('shop_jack', 'Лавка', 'handleShopJack').

Регистрация из JSON-плагинов повторяется при каждой загрузке game_data (в игре — в applyGameData). Свои register() вызывайте после загрузки данных или в том же скрипте до первого визита на special-сцену.

29.4 Если обработчик не найден

Движок покажет предупреждение с подсказкой, кнопку «показать как обычную сцену» (если в редакторе заполнены текст/выборы) и «Назад». Чёрного экрана не будет.

Без special: оставьте поле пустым — сцена работает как обычная.

§30. Plugin API для special-сцен

Движок поддерживает расширение функциональности без правки ядра (engine.js). Вы можете добавлять собственные механики (мини-игры, алхимию, уникальные магазины) двумя способами: через JSON или через внешний JavaScript.

30.1. JSON-плагины (без кода)

Для простых сцен с текстом, выборами и флагами используйте секцию plugins.specialScenes в game_data.json:

{
  "plugins": {
    "specialScenes": {
      "dice_game": {
        "label": "Мини-игра в кости",
        "text": "Трактирщик предлагает сыграть. Ставка — 10 золотых.",
        "dialogue": [
          { "speaker": "Трактирщик", "text": "Ну что, рискнёшь?" }
        ],
        "choices": [
          { "text": "🎲 Играть (10 зм)", "to": "dice_roll", "showIf": { "goldMin": 10 } },
          { "text": "← Отказаться", "to": "tavern" }
        ]
      }
    }
  }
}

JSON-плагины автоматически регистрируются при загрузке данных. ID плагина должен совпадать с полем special у сцены.

30.2. JS-плагины (расширенная логика)

Для сложной логики (расчёты, циклы, обращение к состоянию игры) создайте отдельный JS-файл и подключите его после special-scenes.js:

<script src="js/special-scenes.js"></script>
<script src="plugins/my-custom-plugin.js"></script>

Регистрация обработчика:

SpecialSceneRegistry.register('alchemy_station', 'Алхимический станок', function(engine, sceneId, scene) {
  // Доступ к состоянию игры
  const gold = engine.state.gold;
  const hasHerb = engine.state.inventory.includes('herb');

  // Установка текста и диалогов
  engine.setText('Старый стол с колбами. ' + (hasHerb ? 'У вас есть ингредиенты.' : 'Нужны травы.'));
  engine.setDialogue([
    { speaker: 'Алхимик', text: 'Смешать зелье лечения? 5 золотых.' }
  ]);

  // Формирование выборов динамически
  const choices = [];
  if (hasHerb && gold >= 5) {
    choices.push({
      text: '🧪 Смешать зелье (5 зм)',
      action: 'alchemy_craft'  // Можно использовать action для вызова других методов
    });
  }
  choices.push({ text: '← Уйти', to: 'village' });
  engine.setChoices(choices);
});

§31. Создание собственных систем правил

Движок поддерживает несколько RPG-систем через абстракцию RuleSystem. Вы можете создать собственную систему (например, упрощённую d6, FATE, GURPS) без правки ядра.

31.1. Наследование от RuleSystem

Создайте файл js/systems/my-system.js:

class MyD6System extends RuleSystem {
  get id() { return 'my_d6'; }
  get label() { return 'Моя d6-система'; }
  get description() { return 'Упрощённая система на кубиках d6.'; }

  // Переопределяем модификатор: d6 не использует (score-10)/2
  getModifier(score) {
    return Math.floor((Number(score) - 3) / 3); // Пример: 3-5=0, 6-8=1, 9+=2
  }

  // Point Buy или другая генерация характеристик
  getPointBuyConfig() {
    return { total: 12, min: 1, max: 6 }; // Распредели 12 очков, значения 1-6
  }

  // Расчёт ОЗ
  calculateHP(classKey, level, stats, data, conMod) {
    const base = data?.classes?.[classKey]?.hp || 6;
    return base + (conMod || 0) * Math.max(1, level);
  }

  // Расчёт защиты
  calculateAC(stats, equipment, data, engine) {
    const dexMod = this.getModifier(stats?.dex ?? 3);
    return 3 + dexMod; // Базовая защита 3 + мод. ЛОВ
  }

  // Бросок атаки
  rollAttack(attacker, target, engine) {
    const roll = engine.d(6);
    const total = roll + (attacker.atkBonus || 0);
    const ac = target?.ac || 3;
    const hit = total >= ac;
    const crit = roll === 6;
    let dmg = 0;
    if (hit) {
      dmg = engine.parseRoll(attacker.dmgRoll || '1d6') + (attacker.dmgBonus || 0);
      if (crit) dmg *= 2;
    }
    return { roll, total, hit, crit, dmg };
  }
}

const MySystem = new MyD6System();
if (typeof window !== 'undefined') window.MyD6System = MySystem;

31.2. Регистрация системы

Подключите файл после registry.js и зарегистрируйте:

<script src="js/systems/base-system.js"></script>
<script src="js/systems/dnd5e.js"></script>
<script src="js/systems/pathfinder2e.js"></script>
<script src="js/systems/my-system.js"></script>
<script src="js/systems/registry.js"></script>
<script>
  if (typeof SystemRegistry !== 'undefined' && typeof MyD6System !== 'undefined') {
    SystemRegistry.register(MyD6System);
  }
</script>

После регистрации система появится в выпадающем списке «Система правил» при создании нового проекта в редакторе.
